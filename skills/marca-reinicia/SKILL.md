## Identidad visual — Tipografía

### Tipografía principal: Manrope

- **Descarga:** Google Fonts (tipografía libre)
- **Pesos utilizados:** 300 (Light), 400 (Regular), 700 (Bold)
- **Cursiva/itálica:** la tipografía no dispone de cursiva nativa; **no se utiliza**
- **Ligaduras:** tiene ligaduras estilísticas interesantes activables; consultar la web oficial de Manrope

### Tipografía alternativa: Verdana

Usar cuando Manrope no esté disponible (p.ej. en herramientas online que no soporten fuentes personalizadas). Es una tipografía web-safe humanista y geométrica, parecida en carácter a Manrope.

### Bug conocido: fallback Manrope → Roboto en herramientas Zoho

⚠️ **Caso especial validado en Zoho Sheet, Zoho Writer y Zoho Show (mayo 2026).**

**Síntoma**: aplicar `font_name: "Manrope"` mediante la API (`ZohoSheet_format_ranges`, ranges de Writer, etc.) devuelve `status: success`, pero al abrir el documento algunas (o todas) las celdas se renderizan en **Roboto** en lugar de Manrope. El cambio es **silencioso** — no hay error ni warning.

**Causa**: Zoho hace fallback automático a Roboto cuando Manrope **no está en el catálogo personal de fuentes** del usuario propietario del workbook/documento. La API acepta el nombre Manrope porque es válido conceptualmente, pero el render usa fuente alternativa.

**Decisión de marca**: en herramientas Zoho **NO** se acepta el fallback a Roboto como sustituto canónico. Manrope debe verse en el documento final.

**Workaround operativo**:

1. **Una vez por usuario propietario**: añadir Manrope al catálogo personal de fuentes de Zoho desde la UI: `Configuración Zoho > Fuentes personalizadas > Añadir fuente > Manrope` (descargar desde Google Fonts si hace falta).
2. **En cada generación de documento**: la skill sigue enviando `font_name: "Manrope"` siempre — no hay cambio en el código.
3. **Validación obligatoria al cierre**: antes de marcar un documento como listo para envío al cliente, **verificación visual** de que la tipografía es Manrope (mirar 2-3 celdas/párrafos de cuerpo y cabeceras). Si aparece Roboto, aplicar workaround del paso 1 y reaplicar formato.
4. **Si el usuario propietario no puede / no quiere añadir Manrope**: fallback aceptable únicamente en este caso a **Verdana** (no Roboto). Documentar la decisión en el documento.

**Skills que aplican este workaround**:
- `informes-dedicacion-clientes-reinicia` (validado mayo 2026, Gonher)
- `revision-sprint-backlog-equipo-reinicia` (Sprint Backlogs AUTOIA)
- `informes-ejecutivos-sprint-backlog-equipos-reinicia`
- `plan-proyecto-zoho-sheet-reinicia`
- Cualquier skill futura que genere documentos en Zoho Sheet / Writer / Show

**Lo que NO hacer**:
- ❌ Aceptar Roboto silenciosamente como "casi Manrope" — no lo es y rompe la consistencia visual de marca.
- ❌ Cambiar `font_name` a Roboto en el código de la skill — el dato canónico sigue siendo Manrope.
- ❌ Confundir este caso con el fallback genérico a Verdana documentado más arriba (que aplica a herramientas que NO soportan fuentes personalizadas, no a Zoho).

### Escala tipográfica (ofiмática / documentos)

La escala oficial usa una relación de **cuarta aumentada** (×1.414 entre niveles). Para documentos de oficina del día a día, los tamaños validados son:

| Uso | Fuente | Tamaño | Peso | Color |
|-----|--------|--------|------|-------|
| Encabezado / H1 | Manrope | **36 pt** (sz: 72 en docx-js) | 400 Regular | `#0D0D0D` |
| Subtítulo / H2 | Manrope | **18 pt** (sz: 36 en docx-js) | 700 Bold | `#0D0D0D` |
| Cuerpo de texto | Manrope | **12 pt** (sz: 24 en docx-js) | 400 Regular | `#545454` |
| Cuerpo mínimo | Manrope | **10 pt** | 400 Regular | `#545454` |
| Nota / pie | Manrope | ~10 pt | 400 Regular | `#888888` (gris suave) |

### Usos tipográficos

- **Títulos principales:** color negro `#000000` o `#0D0D0D`, peso 300 (Light) para los más importantes, peso 400 para el resto
- **Párrafos:** color gris oscuro `#545454`, interlineado 1,5× el tamaño del texto
- **Listados:** ligadura `->` (que crea una flecha) o número con guión bajo después; espaciado de una línea entera entre cada elemento
- **Destacados:** sombreado amarillo (`#EBE31B` / versión suave `#FEFDCD`) o cambio de color a azul
- **Cabeceras de tabla:** fondo azul Reinicia `#3812CF`, texto blanco en negrita, separadores blancos gruesos
- **Bloques de empresa en tabla:** fondo lila `#D9D0FB`

### Recursos tipográficos de marca

- **Símbolos de camino:** el guión bajo `_` al final de frases principales actúa como signo de puntuación de marca (*"Te enseñamos el camino inteligente para alcanzar tu meta_"*)
- **Flecha de marca:** `->` (guión + mayor que) para indicar elementos de interés o pasos: *"Paso 1 > Paso 2 > Paso 3"*, *"-> Elemento de interés"*

### Patrón 2 — Tabla estándar Reinicia

Función polivalente `tCell` con cuatro variantes (`header`, `block`, `alt`, normal) más ejemplo de uso. La cabecera principal es lila `#D9D0FB`, las filas alternan blanco y gris `#EBEBEB`, los bloques de empresa usan lila también.

```javascript
import { TableCell, Paragraph, TextRun, BorderStyle, ShadingType, WidthType } from "docx";

/**
 * Construye una celda de tabla estándar Reinicia.
 * @param {string} text - Contenido textual.
 * @param {object} opts - Opciones de variante.
 * @param {boolean} opts.header - Cabecera principal (fondo lila, texto en bold).
 * @param {boolean} opts.block  - Bloque de empresa o destacado (fondo lila).
 * @param {boolean} opts.alt    - Fila alterna (fondo gris #EBEBEB).
 * @param {boolean} opts.bold   - Texto en negrita.
 */
function tCell(text, opts = {}) {
  const fill = opts.header ? LILA
             : opts.block  ? LILA
             : opts.alt    ? GRIS_FILA_ALTERNA
             : BLANCO;

  const color = (opts.header || opts.block) ? NEGRO_TITULAR : GRIS_OSCURO;
  const bold  = opts.header || opts.block || opts.bold || false;

  return new TableCell({
    shading: { type: ShadingType.CLEAR, fill, color: "auto" },
    borders: {
      // Bordes en blanco gruesos — nunca grises
      top:    { style: BorderStyle.SINGLE, size: 8, color: BLANCO },
      bottom: { style: BorderStyle.SINGLE, size: 8, color: BLANCO },
      left:   { style: BorderStyle.SINGLE, size: 8, color: BLANCO },
      right:  { style: BorderStyle.SINGLE, size: 8, color: BLANCO },
    },
    children: [new Paragraph({
      children: [new TextRun({
        text, font: FUENTE, size: 22, color, bold,
      })],
    })],
  });
}

// Ejemplo de uso — tabla con cabecera + filas alternas
const tabla = new Table({
  width: { size: 100, type: WidthType.PERCENTAGE },
  rows: [
    new TableRow({ children: [
      tCell("Concepto",  { header: true }),
      tCell("Valor",     { header: true }),
      tCell("Estado",    { header: true }),
    ]}),
    new TableRow({ children: [
      tCell("Fila 1"), tCell("100"), tCell("OK"),
    ]}),
    new TableRow({ children: [
      tCell("Fila 2", { alt: true }),
      tCell("200",    { alt: true }),
      tCell("OK",     { alt: true }),
    ]}),
  ],
});
```

**Lo que NO usar en tablas:**
- ❌ Bordes en gris (todos los bordes deben ser **blancos gruesos** para crear el efecto de separación limpia entre celdas).
- ❌ Filas todas blancas sin alternancia (siempre alternar `#FFFFFF` con `#EBEBEB`).
- ❌ Cabeceras con fondos saturados de marca (`#3812CF` o `#D14351`) — usar lila `#D9D0FB`.

### Patrón 3 — Chips de prioridad / estado

Función genérica `chipCell` que recibe un mapa de niveles a configuración de colores. Reutilizable para cualquier conjunto de chips: prioridad, estado, fases de proyecto, niveles de riesgo, tipos de evento, etc.

```javascript
import { TableCell, Paragraph, TextRun, BorderStyle, ShadingType, AlignmentType } from "docx";

/**
 * Construye una celda de chip de marca Reinicia.
 * @param {string} nivel - Etiqueta del chip (ej: "CRÍTICO", "ACUERDO").
 * @param {object} mapa  - Objeto que mapea niveles a {fill, text}.
 */
function chipCell(nivel, mapa) {
  const cfg = mapa[nivel] || mapa.DEFAULT || { fill: GRIS_CLARO, text: GRIS_OSCURO };
  return new TableCell({
    shading: { type: ShadingType.CLEAR, fill: cfg.fill, color: "auto" },
    borders: {
      top:    { style: BorderStyle.SINGLE, size: 8, color: BLANCO },
      bottom: { style: BorderStyle.SINGLE, size: 8, color: BLANCO },
      left:   { style: BorderStyle.SINGLE, size: 8, color: BLANCO },
      right:  { style: BorderStyle.SINGLE, size: 8, color: BLANCO },
    },
    children: [new Paragraph({
      alignment: AlignmentType.CENTER,
      children: [new TextRun({
        text: nivel, font: FUENTE, size: 20, color: cfg.text, bold: true,
      })],
    })],
  });
}

// Mapa de ejemplo 1 — Prioridad
```

### Reglas negativas (qué NO hacer)

Lista consolidada de lo que **nunca** debe aparecer en un documento `.docx` de marca Reinicia:

1. ❌ **Bordes grises en tablas** — siempre blancos gruesos (size 8 en `docx-js`) para crear separación limpia.
2. ❌ **Filas todas blancas sin alternancia** — alternar `#FFFFFF` con `#EBEBEB` para legibilidad.
3. ❌ **`#EBE31B` (amarillo puro) como fondo de chip con texto encima** — chirría con el texto y compite con la jerarquía. Usar `#FEFDCD` (Yellow 0131 C) como fondo de pastilla; reservar `#EBE31B` para sombreado puntual de palabras clave en cuerpo de texto.
4. ❌ **Saturados (`#3812CF`, `#D14351`, `#70EED6`) como fondo de chip** — son colores de acento, no de pastilla. La regla conservadora reserva fondos pálidos para chips.
5. ❌ **Inventar colores fuera de paleta** — cualquier color que no aparezca en la paleta canónica documentada arriba (principales + secundarios + colores de recurso) no debe usarse. Si surge una necesidad nueva, mapear a la paleta existente; si no encaja, plantearlo como cambio de marca, no resolverlo localmente en el documento.
6. ❌ **Cursivas en Manrope** — la tipografía no dispone de cursiva nativa. Usar negrita o cambio de color para énfasis.
7. ❌ **Generar el logo sintéticamente** — siempre extraerlo de Workdrive según el procedimiento documentado en la sección "Logo corporativo".
